Introducción



Para este tutorial hemos configurado un entorno compuesto por dos máquinas víctimas. Tras comprometer la primera máquina, necesitaremos establecer un túnel que nos permita acceder a la segunda, donde finalmente obtendremos la flag de administrador.



Este proceso incluye técnicas de recolección de información, explotación, post-explotación, tunneling, port forwarding y pivoting.



Recolección de Información



Comenzamos realizando un primer escaneo con Nmap para identificar puertos y servicios disponibles.







El segundo escaneo, más completo, revela varios vectores de ataque: SSH, HTTP y SMB, además de la dirección web:



scanthenet.labs.local








Para resolver el dominio añadimos su IP al archivo:



/etc/hosts








Una vez resuelta la ip, aceptaremos el riesgo de encontrarnos en una red desconocida y sin certificado se seguridad SSL.











Al acceder al sitio encontramos la página por defecto de XAMPP. Sin embargo, en el código fuente alguien dejó anotadas las credenciales pass/admin, lo que constituye una fuga de información (information leakage).











Explotación de la Primera Máquina



Probamos las credenciales descubiertas en el servicio SSH, obteniendo acceso exitosamente.







Tras la intrusión realizamos enumeración básica de directorios y posteriormente abrimos una terminal PowerShell para continuar la recolección de información.







Enumeración Avanzada en Windows 11



Uso de winPEAS



Descargamos winPEAS desde nuestro servidor local, pero cada ejecución es bloqueada o eliminada por Windows Defender.











Probamos:




Elevar privilegios de PowerShell



Cambiar políticas de ejecución



Ejecutar la versión .exe y .ps1








Finalmente, optamos por excluir el directorio usado del análisis de Windows Defender (solo en entornos de laboratorio).



Le estamos pidiendo a Windows Defender que no analice, bloquee o intervenga sobre los archivos que se ejecuten desde esta carpeta







La versión .ps1 funciona, pero arroja errores al enumerar directorios, por lo que optamos por una enumeración manual más discreta.











Localizamos la flag de usuario, la enviamos a Kali y la decodificamos de Base64.







Continuamos enumerando correos y encontramos información sensible:








Solo Administrator puede iniciar sesión remotamente.



Metodología para la generación de claves corporativas.








También descubrimos la red oculta 11.11.11.0/24, inaccesible desde Kali pero visible desde Windows 11.







Mediante ARP, identificamos la IP de la máquina oculta.







Descubrimiento de la Red Oculta



Revisamos posibles archivos de registro de logins, pero no aparecen. Sin embargo, un correo menciona claves para rDesktop, asociado al puerto 3389.







Comprobamos el servicio y confirmamos su disponibilidad. Nos damos cuenta que tiene el puerto 5895 abierto y el servicio wsman corriendo en el 5895, que nos indica que el protocolo WinRM esta disponible, por establilidad y sencillez cambiamos de puerto. 







A partir de aquí iniciamos la fase de pivoting.



Post-Explotación: Chisel, Túneles y Pivoting



Descargamos Chisel desde su repositorio y enviamos el ejecutable a la máquina Windows 11.











Usamos Chisel para las tres fases clave:



1. Tunneling



Establecemos un túnel entre Kali (servidor) y Windows 11 (cliente) mediante el puerto 8080.







2. Port Forwarding



Chisel tunela la conexión y redirige:




Entrada: 10.10.10.10:8080



Puerto local remoto: R:5895



Objetivo final: 11.11.11.11:5895








3. Pivoting



El tráfico queda redirigido a través del túnel y ya podemos interactuar con la máquina oculta como si estuviéramos dentro de su red.







Aqui os pongo un esquema general de como se organiza la red.







Explotación Final del Servidor Windows



Al tener establecida la conexión con el servidor, podemos ejecutar comandos sobre este, de igual forma que si se encontrase dentro de nuestra rama de LAN.







Con acceso al puerto remoto, generamos un diccionario específico siguiendo la metodología encontrada, usando un script propio:

Glosario



WS-Management (WS-Man): Protocolo estándar para la administración remota y la interoperabilidad entre sistemas de diferentes operadores. En Windows, el servicio que implementa este protocolo se llama&nbsp;WinRM&nbsp;(Windows Remote Management)&nbsp;y permite a los administradores ejecutar comandos de PowerShell en máquinas remotas de forma segura. El servicio WinRM escucha las solicitudes en los puertos 5985 (HTTP) y 5986 (HTTPS)



Nmap: Herramienta para el escaneo de red y detección de puertos/servicios.



winPEAS: Script de enumeración para Windows usado para identificar vectores de escalada de privilegios.



PowerShell: Shell avanzada y lenguaje de scripting de Windows.



Chisel: Herramienta de tunelización TCP/UDP sobre HTTP útil para pivoting.



CrackMapExec (CME): Herramienta para post-explotación y ataques sobre SMB, WinRM, MSSQL.



Evil-WinRM: Shell interactiva para conectarse mediante WinRM.



ARP: Protocolo usado para mapear direcciones IP a MAC en redes locales.



WinRM: Servicio de administración remota de Windows.



NLA: Network Level Authentication, capa de seguridad previa al inicio de sesión RDP.



Pivoting: Técnica para usar una máquina comprometida como salto hacia otras redes.



Port Forwarding: Redirección de tráfico desde un puerto a otro, local o remoto.



Tunneling: Creación de canales encapsulados para transportar tráfico a redes inaccesibles.



Nmap



Herramienta de escaneo de red que permite identificar puertos, servicios y sistemas operativos.



winPEAS



Script de enumeración para Windows que identifica configuraciones inseguras y posibles rutas de escalada de privilegios.



PowerShell



Shell avanzada de Windows que permite automatizar tareas y ejecutar comandos administrativos.



Chisel



Herramienta de tunelización TCP sobre HTTP que permite realizar port forwarding y pivoting entre redes.



CrackMapExec (CME)



Herramienta de post-explotación diseñada para moverse lateralmente, enumerar sistemas y realizar ataques sobre servicios como SMB, WinRM, MSSQL.



evil-winRM



Shell remota para interactuar con WinRM, utilizada frecuentemente para post-explotación en sistemas Windows.



ARP



Protocolo que permite mapear direcciones IP a direcciones MAC dentro de una red local.



Base64



Método de codificación que permite representar datos binarios en texto ASCII.
